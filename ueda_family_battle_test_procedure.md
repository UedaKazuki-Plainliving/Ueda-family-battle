# テスト手順書：大乱闘 ウエダファミリー

**文書バージョン:** 1.0  
**作成日:** 2026-04-22  
**対応仕様書:** ueda_family_battle_test_spec.md  
**テスト前提:** ブラウザで HTML ファイルをローカルで開いて手動操作する

---

## 共通前提条件

- デスクトップ版テストは `C:\Users\plain\street-brawler.html` を Chrome または Edge で開いて実施する
- モバイル版テストは `C:\Users\plain\street-brawler-mobile.html` をモバイルブラウザまたはデバイスエミュレーターで開いて実施する
- 音響テストはブラウザの音量をオンにし、スピーカーまたはイヤホンを使用する
- 各テスト実施前はページをリロード（F5）して初期状態に戻すこと

## TC-001 タイトル画面表示

**対象機能:** タイトル画面表示（デスクトップ版・モバイル版共通）  
**前提条件:** HTML ファイルをブラウザで開いた直後  
**手順:**
1. `street-brawler.html` をブラウザのアドレスバーにドラッグ＆ドロップ、またはダブルクリックして開く
2. タイトル画面の表示内容を目視確認する

**期待結果:**
- 「大乱闘　ウエダファミリー」が黄色・大きなフォントで表示される
- 「対戦格闘ゲーム　全5キャラ」のサブテキストが表示される
- デスクトップ版: 「SPACE か ENTER でスタート」が点滅表示される
- モバイル版: 「タップしてスタート」が点滅表示される

**確認ポイント:** タイトルロゴが見切れていないこと、点滅テキストがおよそ 0.5秒ごとに切り替わること

## TC-002 タイトルからキャラ選択への遷移（デスクトップ）

**対象機能:** ゲームフロー遷移  
**前提条件:** タイトル画面が表示されている  
**手順:**
1. タイトル画面が表示されていることを確認する
2. Space キーを押す

**期待結果:**
- キャラクター選択画面に切り替わる
- 「あなたのキャラを選択」の見出しが表示される
- 5つのキャラパネルが横並びで表示される

**確認ポイント:** Enter キーを押した場合も同様に遷移すること（両キーで試すこと）

## TC-003 タイトルからキャラ選択への遷移（モバイル）

**対象機能:** ゲームフロー遷移（モバイル）  
**前提条件:** モバイル版タイトル画面が表示されている  
**手順:**
1. モバイルブラウザで `street-brawler-mobile.html` を開く
2. タイトル画面の「タップしてスタート」が表示されていることを確認する
3. 画面（キャンバス）を 1回タップする

**期待結果:**
- キャラクター選択画面に遷移する
- 「あなたのキャラを選択」「タップして選択 / 決定で確定」が表示される

**確認ポイント:** タップが反応するためには AudioContext の resume が必要。タップが受け付けられない場合は再度タップすること

## TC-004 ラウンド開始演出

**対象機能:** ラウンド開始フロー  
**前提条件:** キャラクター選択画面が表示されている  
**手順:**
1. キャラ選択画面でプレイヤーキャラを選択し確定する（Space/Enter）
2. CPU キャラを選択し確定する（Space/Enter）
3. 対戦画面に切り替わったタイミングを観察する

**期待結果:**
- 「ROUND 1」フラッシュが大きく表示される
- 約 1.2秒後に「FIGHT!」フラッシュが表示される
- 約 1.8秒後にフラッシュが消えて対戦操作が可能になる
- キャラクターが左右に対峙して立っている

**確認ポイント:** 演出中はプレイヤーの操作が受け付けられないこと（countdown フェーズ）

## TC-005 KO によるラウンド終了

**対象機能:** ラウンド勝敗判定（KO）  
**前提条件:** 対戦中（fight フェーズ）  
**手順:**
1. 対戦を開始する
2. プレイヤーが攻撃を当て続けて相手（CPU）の HP を 0 にする
3. または CPU にわざと攻撃を受けて自分の HP を 0 にする
4. ラウンド終了時の表示を確認する

**期待結果:**
- プレイヤーが勝った場合: 「[キャラ名] WIN!」が画面中央に黄色で表示される
- CPU が勝った場合: 「CPU WIN!」が画面中央に表示される
- 対応ラウンド勝利ドット（HUD 下部の丸）が 1個追加点灯する

**確認ポイント:** HP が 0 になった瞬間ではなく、フレーム処理で検知されるため若干のラグがある場合がある

---

## TC-006 タイムアウトによるラウンド終了

**対象機能:** ラウンド勝敗判定（タイムアウト）  
**前提条件:** 対戦中（fight フェーズ）  
**手順:**
1. 対戦を開始し、両プレイヤーが攻撃しない状態（またはガードし続ける状態）で待機する
2. タイマーが 0 になるのを待つ（最大 99秒）
3. タイムアウト後の表示を確認する

**期待結果:**
- timerSec が 0 になるとラウンドが終了する
- HP が多い方が勝者として「[キャラ名] WIN!」または「CPU WIN!」が表示される
- HP が同値なら「DRAW!」が表示される

**確認ポイント:** タイマーは画面中央上部に表示。残り 10秒で赤色に変わることも合わせて確認すること

---

## TC-007 引き分け

**対象機能:** 引き分け判定  
**前提条件:** タイムアウト時に両者 HP が同値になる状況  
**手順:**
1. TC-006 の手順でタイムアウトを発生させる
2. タイムアウト時に両プレイヤーの HP バーがほぼ同じ長さになっていることを確認する（ゲームが自動で引き分けと判定する）

**期待結果:**
- タイムアウト時に両者 HP が完全に等値の場合「DRAW!」フラッシュが表示される

**確認ポイント:** CPU が動く関係上、完全な引き分けを意図的に再現するのは困難。発生した場合に確認する

## TC-008 3ラウンド先取での試合終了

**対象機能:** ゲームフロー（試合終了・リセット）  
**前提条件:** 対戦進行中  
**手順:**
1. 対戦を開始し、2ラウンド連続でプレイヤーが（または CPU が）勝利する
2. 2ラウンド目の勝利後の画面遷移を観察する

**期待結果:**
- 2ラウンド勝利した側の WIN! フラッシュが表示される
- 約 3〜4秒後にキャラクター選択画面（selectStep=0）に自動で戻る
- 勝利ドットがリセットされ、ラウンド番号が 1 に戻る

**確認ポイント:** 3ラウンド目が始まらず、選択画面に戻ることを確認する

## TC-009 ラウンド2への自動移行

**対象機能:** ラウンド移行  
**前提条件:** 1ラウンド終了（どちらかが 1勝した状態）  
**手順:**
1. 対戦を開始し、1ラウンドを終了させる（KO またはタイムアウト）
2. ラウンド終了後に自動で次ラウンドが開始されるまで待機する

**期待結果:**
- endTimer（約 200フレーム ≒ 3.3秒）経過後に「ROUND 2」フラッシュが表示される
- 続けて「FIGHT!」フラッシュが表示され、2ラウンド目が開始される
- 両キャラクターが初期位置（左: x=200、右: x=600）に戻る

**確認ポイント:** HP・特殊ゲージが初期状態にリセットされていること

## TC-012 キャラ変更（デスクトップ・キーボード）

**対象機能:** キャラ選択操作  
**前提条件:** キャラクター選択画面（ステップ 1）が表示されている  
**手順:**
1. → キー（または D キー）を複数回押す
2. カーソルが右に動くことを確認する
3. 「おかあさん」にカーソルがある状態で → キーをさらに押す
4. ← キー（または A キー）を押してカーソルを左に戻す

**期待結果:**
- → キーで選択中のキャラが右隣に移動する
- 右端（おかあさん）で → を押すと左端（いっちー）に折り返す（循環）
- ← キーで左隣に移動する
- カーソル位置のパネルが点滅ハイライトされ、「▶ あなた」が表示される

**確認ポイント:** カーソル移動時に SE が再生されること

## TC-017 左右移動（デスクトップ）

**対象機能:** プレイヤー移動  
**前提条件:** 対戦中（fight フェーズ）  
**手順:**
1. 対戦を開始する
2. A キー（または ← キー）を押しっぱなしにする
3. 移動が確認できたらキーを離し、今度は D キー（または → キー）を押しっぱなしにする

**期待結果:**
- A キー: プレイヤーが左方向に歩く（足踏みアニメーション付き）
- D キー: プレイヤーが右方向に歩く
- キーを離すと停止してアイドル状態に戻る

**確認ポイント:** 移動速度が各キャラの walkSpd 設定に応じて異なること（いっちー: 6.5、おとうさん: 2.0 など）

## TC-018 ジャンプ（デスクトップ）

**対象機能:** ジャンプ  
**前提条件:** 対戦中、プレイヤーが地上にいる  
**手順:**
1. W キー（または ↑ キー）を押す
2. プレイヤーが空中に浮かぶことを確認する
3. 着地するまで観察する

**期待結果:**
- W キーでキャラが飛び上がる（jumpVel 分の初速で上昇、gravity で落下）
- 放物線を描いて GROUND（y=340）に着地する
- 着地後 idle 状態に戻る

**確認ポイント:** キャラごとにジャンプ高さが異なること（そよが最も高く、おとうさんが低め）

3. 勝利ドットの変化を確認する

- 2個ドットが点灯すると試合終了（TC-008 参照）

**確認ポイント:** 左側（x=55, 73付近）が P1のドット、右側（x=745, 727付近）が P2（CPU）のドット

## TC-057 攻撃ヒット SE

**対象機能:** 音響（攻撃 SE）  
**前提条件:** ブラウザの音量オン、対戦中  
**手順:**
1. 対戦中に攻撃（パンチ・キック）を CPU に当てる
2. SE が再生されるか確認する

**期待結果:**
- 通常攻撃ヒット: 高音の「ビシュ」SE（420〜600Hz の square 波、短時間）が再生される
- 必殺技ヒット（big=true）: より低音で大きい SE が再生される

**確認ポイント:** SE が無音でないこと。ブラウザの AutoPlay ポリシーにより、最初のユーザー操作後に AudioContext が有効になること

**前提条件:** モバイルブラウザで対戦中  
**手順:**

## TC-060 コントロール表示の切り替え（モバイル）

**対象機能:** モバイルの UI 切り替え  
**前提条件:** モバイル版でゲームが動作している  
**手順:**
1. タイトル画面・キャラ選択画面を表示中に nav-ctrls（◀ 決定 ▶）が表示されていることを確認する
2. 対戦が開始されたとき（fight フェーズ）に fight-ctrls（十字パッド＋アクションボタン）が表示されることを確認する
3. ラウンド終了後（roundend フェーズ）も fight-ctrls が表示されたままであることを確認する

**期待結果:**
- タイトル・キャラ選択: nav-ctrls 表示、fight-ctrls 非表示（display: none）
- 対戦中・ラウンド終了・カウントダウン: fight-ctrls 表示、nav-ctrls 非表示

**確認ポイント:** setInterval（80ms ごと）でフェーズを監視して自動切り替えされること

## TC-061 アクションボタン動作確認（モバイル）

**対象機能:** モバイルのアクションボタン  
**前提条件:** モバイル版で対戦中、fight-ctrls が表示されている  
**手順:**
1. 「パンチ（👊）」ボタンを長押しする → CPU の近くで当てる
2. 「キック（🦵）」ボタンを長押しする → CPU の近くで当てる
3. 「ガード（🛡）」ボタンを長押しする → CPU の攻撃を受ける
4. ゲージが MAX のときに「必殺技（⚡）」ボタンをタップする

**期待結果:**
- パンチボタン: KeyJ が keys に設定され、地上パンチが発動する
- キックボタン: KeyK が keys に設定され、地上キックが発動する
- ガードボタン: KeyL が keys に設定され、立ちガード状態になる
- 必殺技ボタン: Space が keys に設定され、ゲージ MAX なら必殺技が発動する

**確認ポイント:** 各ボタンが長押し中に active クラスが付き、ボタンが明るく表示されること

**対象機能:** タッチ操作のスクロール抑制

**手順:**
