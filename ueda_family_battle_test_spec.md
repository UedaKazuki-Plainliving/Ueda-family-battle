## 2. テスト観点一覧

### 2.1 ゲームフロー

- タイトル画面（イントロ）の表示と遷移
- キャラクター選択画面の操作性
- ラウンド開始演出（ROUND X 表示・FIGHT! 表示）
- ラウンド勝敗判定（KO / タイムアウト / 引き分け）
- ラウンドウィン数のカウントと3ラウンド先取での試合終了
- 試合終了後のキャラ選択画面への復帰

### 2.2 キャラクター選択

- 5キャラクター（いっちー・ゆうり・そよ・おとうさん・おかあさん）の全選択
- プレイヤーキャラ選択 → CPU キャラ選択の2ステップ遷移
- 選択中キャラの強調表示（点滅ハイライト）
- 決定後の自プレイヤーキャラへの「チェックマーク」表示
- デスクトップ: キーボード（←/→）による選択、Space/Enter/Jでの確定
- モバイル: タップによるキャラ選択、「決定」ボタンによる確定

### 2.3 基本操作・移動

- 左右移動（歩き）
- ジャンプ・斜めジャンプ
- しゃがみ（押しっぱなし）
- ステージ端での移動制限（x: 55〜745の範囲内）
- 相手との密着時の押し返し処理

### 2.4 攻撃判定

- 地上パンチ（ヒット時のダメージ・ヒットストップ・ノックバック）
- 地上キック（ヒット時のダメージ・ヒットストップ・ノックバック）
- 空中パンチ（jatk）のヒット判定
- 空中キック（jkick）のヒット判定
- 攻撃中の判定フレーム（各キャラの punchS〜punchE / kickS〜kickE 範囲内のみ有効）
- ヒット時のエフェクト（火花スプライト）と SE

### 2.5 ガードシステム

- 立ちガード（Lキー / ガードボタン）が地上パンチ・空中パンチ（jatk）を無効化すること
- 立ちガードが地上キック・空中キック（jkick）を防がないこと（ダメージを受ける）
- しゃがみガード（しゃがみ中 + ガードボタン）が地上キック・空中キック（jkick）を無効化すること
- しゃがみガードが地上パンチ・空中パンチ（jatk）を防がないこと
- 単なるしゃがみ（ガードボタンなし）はキックを防がないこと
- ガード成功時: ダメージ 0・guardFlash=14 セット・低音 SE（beep 95Hz + 60Hz）
- 立ちガードエフェクト: 縦弧 + テキスト「カン」
- しゃがみガードエフェクト: 横広弧 + テキスト「しゃがみガード」
- いずれのガードでも必殺技攻撃はダメージ軽減（40%のダメージが通る）
- 背後（ファーシング不一致）からの攻撃はガード不能

### 2.6 特殊攻撃（必殺技）

- 特殊ゲージが MAX（100）になると発動可能（「必殺技 READY!」表示と点滅エフェクト）
- ゲージ未満での Space キー（必殺技ボタン）操作で発動しないこと
- いっちー：ドラゴンソード（飛び道具、dmg=28、スタン 25）
- ゆうり：ドロップキック（突進技、dmg=35、スタン 42）
- そよ：陸上こうげき（ハードル4本、各 dmg=10）
- おとうさん：沢山しゃべる（ワード飛び道具 9発、各 dmg=6）
- おかあさん：ゴリラ攻撃（ゴリラ召喚、dmg=12）
- 必殺技発動時の画面フラッシュ・バナー演出
- 必殺技ヒット時に攻撃側のゲージが増加すること

### 2.7 飛び道具（プロジェクタイル）

- おとうさんのパンチが野球ボールを発射すること（punchIsProjectile=true）
- 飛び道具がステージ端で消滅すること（x < -90 または x > 890）
- 飛び道具がヒット後に消滅すること

### 2.8 CPU AI

- CPU がプレイヤーに向かって接近すること
- CPU が攻撃（パンチ・キック・ジャンプ）を行うこと
- CPU が必殺技ゲージ MAX 時に一定確率で必殺技を使用すること
- CPU がガード行動をとることがあること

### 2.9 HUD・UI

- 両プレイヤーの HP バー表示と残量に応じた色変化（>50%: 緑、>25%: 黄、<=25%: 赤）
- 特殊ゲージバーの表示と増加
- 試合タイマー（99秒カウントダウン、残り10秒以下で赤表示）
- ラウンド勝利数のドット表示（最大2個）
- KO / WIN / CPU WIN / DRAW のフラッシュメッセージ表示

### 2.10 音響

- 攻撃ヒット時の SE
- ガード成功時の SE（低音 2音）
- ジャンプ SE
- 必殺技発動 SE（キャラ別）
- タイトル・選択画面のナビゲーション SE

### 2.11 モバイル固有機能

- 十字パッド（上下左右）によるキャラ操作
- アクションボタン（パンチ・キック・ガード・必殺技）の動作
- イントロ画面でのタップによるスタート
- キャラ選択画面でのキャラパネルタップによる選択
- ナビゲーションボタン（◀ 決定 ▶）による選択・確定
- 戦闘中: fight-ctrls（十字 + アクション）が表示され、nav-ctrls が非表示になること
- キャラ選択中: nav-ctrls が表示され、fight-ctrls が非表示になること
- タッチ操作中のスクロール・ズーム抑制（touch-action: none）
- キャンバスのレスポンシブ表示（width: 100vw、高さ自動）

## 4. 合格基準

### 4.1 必須合格項目（優先度：高）

以下の項目はすべて合格すること。1件でも不合格の場合はリリース不可とする。

- TC-001〜TC-009（ゲームフロー全体の正常動作）
- TC-011〜TC-013、TC-015〜TC-016（キャラクター選択の基本動作）
- TC-017〜TC-019、TC-022（基本操作の動作）
- TC-023〜TC-024、TC-029（攻撃判定の基本動作）
- TC-030〜TC-035（ガードシステム全体）
- TC-038〜TC-045（特殊攻撃の全キャラ正常動作）
- TC-056（勝敗フラッシュメッセージ）
- TC-060〜TC-061（モバイルコントロールの基本動作）

### 4.2 推奨合格項目（優先度：中・低）

優先度「中」はすべて合格、優先度「低」は 80% 以上合格することを推奨する。

### 4.3 既知の不確実性

CPU 動作に関するテスト（TC-050〜TC-052）は確率的動作のため、複数回試行したうえで「概ね仕様通りの傾向が確認できる」ことをもって合格とする。
