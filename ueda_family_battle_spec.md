# 第1部　要件定義書

## 1. プロジェクト概要

### 1.1 タイトル

**大乱闘　ウエダファミリー**

### 1.2 ゲームジャンル・形式

| 項目 | 内容 |
|------|------|
| ジャンル | 2D対戦格闘ゲーム（ストリートブロウラー型） |
| 実装形式 | HTML5 Canvas 2D |
| 対応ファイル | `street-brawler.html`（デスクトップ版）/ `street-brawler-mobile.html`（モバイル版） |
| UI言語 | 日本語 |

### 1.3 プレイヤー人数

- **1人プレイ専用**（対コンピュータAI戦）

---

## 2. 目的・価値提供

### 2.1 ゲームの目的

ウエダファミリーの4名のキャラクターを操作し、コンピュータが操る相手キャラクターと格闘を行う。ラウンド制で先に規定数のラウンドを獲得した側が勝者となる。

### 2.2 プレイヤーへの価値提供

- 家族をモチーフにした親しみやすいキャラクターによる格闘体験
- 4キャラクター各々の異なる能力値・特性により、プレイスタイルの選択肢を提供
- ガードシステム・特殊攻撃ゲージなどの戦略要素による読み合い体験
- デスクトップ・モバイル両対応により、端末を選ばないプレイ環境

---

## 3. 機能要件

### 3.1 必須機能一覧

| ID | 機能名 | 概要 |
|----|--------|------|
| F-01 | キャラクター選択 | 4名（お父さん・お母さん・ゆうり・じゃとく）から1名を選択する |
| F-02 | 対戦フロー制御 | ラウンド開始・終了・勝敗判定・試合終了を管理する |
| F-03 | 移動操作 | 左右移動・しゃがみ・ジャンプを受け付ける |
| F-04 | 攻撃操作 | パンチ・キック・ジャンプ攻撃・ジャンプキック・特殊攻撃を受け付ける |
| F-05 | ガード操作 | 立ちガード・しゃがみガードを受け付け、攻撃を防ぐ |
| F-06 | 攻撃判定 | ヒットボックス・ハートボックスの衝突を検出し、ダメージ・ノックバックを適用する |
| F-07 | HP管理 | 各ファイターのHPを管理し、0になったらラウンド敗北とする |
| F-08 | 特殊攻撃ゲージ | 攻撃の与受によりゲージを蓄積し、一定量でゲージ技を解放する |
| F-09 | タイマー管理 | 1ラウンド60秒を計測し、タイムアウト時はHP優勢側を勝者とする |
| F-10 | CPU AI | 距離・状況に応じて移動・攻撃・ガードを自律判断する |
| F-11 | 音響出力 | Web Audio APIで効果音をリアルタイム生成・再生する |
| F-12 | ガード視覚効果 | ガード成立時に弧形エフェクトとテキストを表示する |
| F-13 | モバイル対応 | タッチボタン入力によりスマートフォン・タブレットでプレイ可能とする |

### 3.2 非機能要件

| ID | 区分 | 要件 |
|----|------|------|
| NF-01 | 動作環境 | モダンブラウザ（Chrome / Firefox / Safari / Edge 最新版）で動作すること |
| NF-02 | 外部依存 | サーバー不要・単一HTMLファイルで完結すること（外部ライブラリ参照なし） |
| NF-03 | 音響 | Web Audio API対応ブラウザにて効果音が再生されること |
| NF-04 | フレームレート | requestAnimationFrameベースで安定した描画を行うこと |
| NF-05 | モバイル操作 | タッチイベントのデフォルト動作をキャンセルし、意図しないスクロールを防ぐこと |
| NF-06 | 保守性 | デスクトップ版・モバイル版で機能を同一に保ち、差異を操作入力のみとすること |

## 4. 主要システム設計

### 4.1 ガードシステム

プレイヤーはガードボタン（またはしゃがみ中ガード）により攻撃を無効化できる。

**ガード成立条件**

| ガード種別 | 成立条件 |
|-----------|---------|
| 立ちガード | `state === 'block'` かつ 相手の方向を向いている（facing） |
| しゃがみガード | `guardMode` かつ `state === 'crouch'` かつ facing |

**guardMode 定義**

```
guardMode = (state === 'block') || (state === 'crouch' && inp.block)
```

**ガードの有効範囲**

| 攻撃の種類 | 立ちガード | しゃがみガード |
|-----------|-----------|-------------|
| パンチ | 有効（防ぐ） | 無効 |
| ジャンプ攻撃（jatk） | 有効（防ぐ） | 無効 |
| キック | 無効 | 有効（防ぐ） |
| ジャンプキック（jkick） | 無効 | 有効（防ぐ） |

> しゃがみ中でもガードボタンを押していない場合、キックを受けるとダメージが入る。

### 4.2 特殊攻撃ゲージシステム

- 攻撃を**与える**または**受ける**ことでゲージが蓄積される。
- ゲージが特殊攻撃消費量以上になると特殊攻撃が使用可能になる。
- 特殊攻撃使用時にゲージを消費する。

### 4.3 CPU AIシステム

- 毎フレーム、距離・自己HP・プレイヤーHP・乱数を元に行動を決定する。
- 近距離では攻撃行動（パンチ・キック・特殊攻撃）を優先する。
- 遠距離では接近行動を取る。
- プレイヤーのHPが低い局面で積極性が高まる傾向がある。
- 一定確率でランダムに行動を変化させることで予測しにくさを実現する。

### 4.4 音響システム

- **Web Audio API**（`AudioContext`）を使用してすべての効果音をリアルタイム生成する。
- 外部音声ファイルには依存しない。
- `beep(frequency, type, volume, duration)` 関数で各効果音を定義する。

### 3.2 腕ポーズ別の描画ロジック

| 状態 | 腕の描画 |
|------|---------|
| しゃがみ（通常） | 腕が自然に垂れ下がる |


## 4. ガードシステム詳細

### 4.1 ガード判定ロジック

```
guardMode = (state === 'block') || (state === 'crouch' && inp.block)

立ちガード成立: guardMode && state === 'block' && facing
しゃがみガード成立: guardMode && state === 'crouch' && facing
```

- **facing**: ファイターが相手の方向を向いているフラグ（方向転換が正確に行われていること）。
- ガードが成立した場合はダメージ = 0 となり、`guardFlash = 8`（フラッシュカウンタ）がセットされる。

### 4.2 ガード視覚エフェクト詳細

| ガード種別 | エフェクト形状 | 表示テキスト | テキスト位置 |
|-----------|-------------|------------|------------|
| 立ちガード | 縦弧（arc -0.72π 〜 +0.72π） | 「カン」 | 弧の中央付近 |
| しゃがみガード | 横広弧（arc -0.85π 〜 +0.85π） | 「しゃがみガード」 | 弧の下 |

- 弧の描画色: `strokeStyle = '#6af'`（青系）

## 5. 音響仕様詳細

### 5.1 beep関数シグネチャ

```
beep(frequency, type, volume, duration)
```

| 引数 | 型 | 説明 |
|------|----|------|
| frequency | number | 周波数（Hz） |
| type | string | 波形タイプ（'sine' / 'square' / 'sawtooth' / 'triangle'） |
| volume | number | 音量（0.0〜1.0） |
| duration | number | 持続時間（秒） |

### 5.2 各効果音パラメータ

| 効果音 | 呼び出し内容 |
|--------|------------|
| 通常ガード（低音その1） | `beep(95, 'sine', 0.28, 0.18)` |
| 通常ガード（低音その2） | `beep(60, 'sine', 0.14, 0.14)` |
| 特殊攻撃ガード | `beep(75, 'sine', 0.22, 0.2)` |

> 攻撃ヒット音・ジャンプ音・ラウンド開始音等はそれぞれ別途 beep 系関数で定義されている。

## 6. 操作仕様詳細

### 6.1 デスクトップ版（キーボード）

| 操作 | キー1 | キー2 |
|------|-------|-------|
| 左移動 | A | ←（Left Arrow） |
| 右移動 | D | →（Right Arrow） |
| しゃがみ | S | ↓（Down Arrow） |
| ジャンプ | W | ↑（Up Arrow） |
| パンチ | J | — |
| キック | K | — |
| ガード（立ち/しゃがみ兼用） | L | — |
| 特殊攻撃 | U | — |

- しゃがみ中に L を押した状態でしゃがみガードが成立する。

### 6.2 モバイル版（タッチボタン）

- 画面上に仮想ボタンを配置し、`touchstart` / `touchmove` / `touchend` / `touchcancel` イベントで入力を検知する。
- `bindHold` 関数により、ボタンを押し続けている間は入力継続状態を維持する。
- すべてのタッチイベントで `preventDefault()` を呼び出し、意図しないスクロール・拡大縮小を防止する。
- 提供ボタン: 移動左・移動右・しゃがみ・ジャンプ・パンチ・キック・ガード・特殊攻撃

## 7. 攻撃判定仕様

### 7.1 ヒット処理フロー

```
1. 攻撃側のヒットボックスと防御側のハートボックスが重なる
2. ガード判定を確認
   a. ガード成立（facing かつ適切なガード種別）
      → ダメージ = 0, guardFlash セット, ガードSE再生
   b. ガード不成立
      → ダメージ適用, ノックバック付与, state → 'hurt', ヒットSE再生, ヒットスパーク表示
3. 特殊攻撃ゲージを更新（攻撃側・防御側ともに蓄積）
```

### 7.2 ノックバック

- ヒット成立時、被ダメージ側は攻撃者と反対方向に一定量押し戻される。
- ノックバック量は攻撃の種類・キャラクターのパラメータに依存する（詳細は実装コードに準拠）。
