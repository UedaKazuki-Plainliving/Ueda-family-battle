# 第1部　要件定義書

## 1. プロジェクト概要

### 1.1 タイトル

**大乱闘　ウエダファミリー**

### 1.2 ゲームジャンル・形式

| 項目 | 内容 |
|------|------|
| ジャンル | 2D対戦格闘ゲーム（ストリートブロウラー型） |
| 実装形式 | HTML5 Canvas 2D |
| 対応ファイル | `street-brawler.html`（デスクトップ版）/ `street-brawler-mobile.html`（モバイル版） |
| UI言語 | 日本語 |

### 1.3 プレイヤー人数

- **1人プレイ**（対コンピュータAI戦）
- **2人プレイ**（対戦相手もキーボードで操作: P2は矢印キー+テンキー）

---

## 2. 目的・価値提供

### 2.1 ゲームの目的

ウエダファミリーの4名のキャラクターを操作し、コンピュータが操る相手キャラクターと格闘を行う。ラウンド制で先に規定数のラウンドを獲得した側が勝者となる。

### 2.2 プレイヤーへの価値提供

- 家族をモチーフにした親しみやすいキャラクターによる格闘体験
- 4キャラクター各々の異なる能力値・特性により、プレイスタイルの選択肢を提供
- ガードシステム・特殊攻撃ゲージなどの戦略要素による読み合い体験
- デスクトップ・モバイル両対応により、端末を選ばないプレイ環境

---

## 4. 主要システム設計

### 4.1 ガードシステム

プレイヤーはガードボタン（またはしゃがみ中ガード）により攻撃を無効化できる。

**ガード成立条件**

| ガード種別 | 成立条件 |
|-----------|---------|
| 立ちガード | `state === 'block'` かつ 相手の方向を向いている（facing） |
| しゃがみガード | `guardMode` かつ `state === 'crouch'` かつ facing |

**guardMode 定義**

```
guardMode = (state === 'block') || (state === 'crouch' && inp.block)
```

**ガードの有効範囲**

| 攻撃の種類 | 立ちガード | しゃがみガード |
|-----------|-----------|-------------|
| パンチ（punch） | 有効（防ぐ・ダメージ0） | 無効（通る） |
| ジャンプ攻撃（jatk） | 有効（防ぐ・ダメージ0） | 無効（通る） |
| キック（kick） | 無効（通る） | 有効（防ぐ・ダメージ0） |
| ジャンプキック（jkick） | 無効（通る） | 有効（防ぐ・ダメージ0） |
| 必殺技（special） | 有効（ダメージ40%に軽減） | 有効（ダメージ40%に軽減） |

> しゃがみ中でもガードボタンを押していない場合、キックを受けるとダメージが入る。  
> 必殺技はどちらのガードでも完全には防げず、40%ダメージが通る。

### 4.2 特殊攻撃ゲージシステム

- 攻撃を**与える**または**受ける**ことでゲージが蓄積される。
- ゲージが特殊攻撃消費量以上になると特殊攻撃が使用可能になる。
- 特殊攻撃使用時にゲージを消費する。

**ゲージ消費タイミング**

| キャラクター | 消費タイミング |
|-------------|--------------|
| ゆうり（ドロップキック） | 特殊攻撃 state 突入と同時に消費（即時発動のため） |
| その他キャラクター | `fireFrame`（実際に技が発動するフレーム）到達時に消費 |

> 特殊攻撃の発動モーションに入っても `fireFrame` に到達する前は技が出ておらず、ゲージを消費しない。

**必殺技無敵フレーム**

- `state === 'special'` の間（特殊攻撃モーション全体）はダメージを受けない無敵状態になる。
- `takeHit()` 関数が `false` を返し、ヒット処理がスキップされる。

### 4.2a コンボブローシステム

- ファイターは `comboCount` プロパティを持ち、連続ヒット数を追跡する。
- 5連続ヒットが成立すると、ノックバック（push）を通常の3倍にして相手を大きく吹き飛ばす。
- 大ノックバック発動後、`comboCount` は 0 にリセットされる。

**comboCount リセット条件**

| タイミング | 内容 |
|-----------|------|
| ガード成立時 | 相手のガードが通るたびにリセット |
| `idle` または `walk` 状態への遷移時 | ファイターが待機・歩行に戻るたびにリセット |

### 4.3 CPU AIシステム

- 毎フレーム、距離・自己HP・プレイヤーHP・乱数を元に行動を決定する。
- 近距離では攻撃行動（パンチ・キック・特殊攻撃）を優先する。
- 遠距離では接近行動を取る。
- プレイヤーのHPが低い局面で積極性が高まる傾向がある。
- 一定確率でランダムに行動を変化させることで予測しにくさを実現する。

### 4.4 音響システム

- **Web Audio API**（`AudioContext`）を使用してすべての効果音をリアルタイム生成する。
- 外部音声ファイルには依存しない。
- `beep(frequency, type, volume, duration)` 関数で各効果音を定義する。

## 4. ガードシステム詳細

### 4.1 ガード判定ロジック

```
guardMode = (state === 'block') || (state === 'crouch' && inp.block)

立ちガード成立: guardMode && state === 'block' && facing
しゃがみガード成立: guardMode && state === 'crouch' && facing
```

- **facing**: ファイターが相手の方向を向いているフラグ（方向転換が正確に行われていること）。
- ガードが成立した場合はダメージ = 0 となり、`guardFlash = 8`（フラッシュカウンタ）がセットされる。

### 4.2 ガード視覚エフェクト詳細

| ガード種別 | エフェクト形状 | 表示テキスト | テキスト位置 |
|-----------|-------------|------------|------------|
| 立ちガード | 縦弧（arc -0.72π 〜 +0.72π） | 「カン」 | 弧の中央付近 |
| しゃがみガード | 横広弧（arc -0.85π 〜 +0.85π） | 「しゃがみガード」 | 弧の下 |

- 弧の描画色: `strokeStyle = '#6af'`（青系）

- 画面上に仮想ボタンを配置し、`touchstart` / `touchmove` / `touchend` / `touchcancel` イベントで入力を検知する。
- `bindHold` 関数により、ボタンを押し続けている間は入力継続状態を維持する。
- すべてのタッチイベントで `preventDefault()` を呼び出し、意図しないスクロール・拡大縮小を防止する。
- 提供ボタン: 移動左・移動右・しゃがみ・ジャンプ・パンチ・キック・ガード・特殊攻撃

**Dパッドの斜めジャンプボタン**

- Dパッド上に **↖（左斜めジャンプ）** および **↗（右斜めジャンプ）** ボタンを追加。
- `bindHold2` 関数を使用し、1回のタッチで「移動」と「ジャンプ」の2キーを同時押し状態にする。

## 7. 攻撃判定仕様

### 7.1 ヒット処理フロー

```
1. 攻撃側のヒットボックスと防御側のハートボックスが重なる
2. 防御側の無敵フレームを確認
   - state === 'special' の場合 → takeHit() が false を返し処理スキップ（無敵）
3. ガード判定を確認
   a. ガード成立（facing かつ適切なガード種別）
      → ダメージ = 0（または40%軽減）, guardFlash セット, ガードSE再生
      → comboCount リセット
   b. ガード不成立
      → ダメージ適用, comboCount インクリメント
      → comboCount が 5 に達した場合、ノックバックを3倍にして comboCount = 0 にリセット
      → ノックバック付与, state → 'hurt', ヒットSE再生, ヒットスパーク表示
4. 特殊攻撃ゲージを更新（攻撃側・防御側ともに蓄積）
```

### 7.2 ノックバック

- ヒット成立時、被ダメージ側は攻撃者と反対方向に一定量押し戻される。
- ノックバック量は攻撃の種類・キャラクターのパラメータに依存する（詳細は実装コードに準拠）。
- **コンボブロー発動時**（5連続ヒット）はノックバック量が通常の3倍になる。
